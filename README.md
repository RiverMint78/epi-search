# epi-search: 定向迭代块搜索

## 1. 核心数据结构

### 表达式表示

算式在程序中以递归树结构表示：

- **`ExprTree`**：一个枚举类型，可以是 `Leaf`（叶子节点，如常数 `e` 或 `pi`）或 `Node`（算子节点，如 `+`, `-`, `*`, `/`, `^` 连接两个子节点）。
- **`MathNode`**：一个结构体，包含 `ExprTree` 及其预计算的 `f64` 数值。
- **`Arc<MathNode>`**：表达式包装在 `Arc`指针中。

## 2. 搜索算法

本程序采用 **定向迭代块搜索（Targeted Iterative Block Search）**。结合小规模 BFS 以及对残差误差的迭代细化。

### A. Block Search

生成深度不超过 `max_ops` 的表达式。

1. **BFS**：从常数（`e`, `pi`）开始，广度构建算式。
2. **目标剪枝**：在每一层中，算式按其与特定 `target` 值的距离进行排序。仅保留最优的 `workspace_size` 个算式到下一层。
3. **线性启发式内存分配**：各层根据线性权重分配内存限制，优先保证深层的搜索多样性。

### B. Iterative Refinement

1. **第一阶段：初始池**：针对用户的 `target` 执行一次块搜索。将前 `top_k` 个结果存入池中。
2. **第二阶段：残差修正**：
   - 对于池中的每个表达式 $E$，计算其误差残差 $R = \text{target} - E.\text{value}$。
   - 为每个 $E$ 发起两次**全新的、定向的**块搜索：
     - **搜索 1**：寻找逼近 $R$ 的算式 $T$。若成功，则 $E + T \approx \text{target}$。
     - **搜索 2**：寻找逼近 $-R$ 的算式 $T$。若成功，则 $E - T \approx \text{target}$。
3. **第三阶段：池进化**：
   - 收集所有 $E \pm T$ 候选算式。
   - 按与目标的距离排序（残差小于0的，字典上优先）。
   - 通过数值去重保持多样性。
   - 截断至 `top_k` 并进入下一次迭代。

## 3. 启发逻辑

### 高精度残差逼近

算法基于这样一个启发式原理：**逼近接近 0 的数值比逼近大数值更容易且精度更高**。

- 在初始阶段后，残差 $R$ 通常已经非常小。
- 针对 $10^{-6}$ 量级的目标搜索修正项 $T$，会发现与针对原始大目标搜索完全不同的（且通常更精准的）算子组合。

### 多样性管理

- **数值去重**：移除数值相同（在极小误差范围内）的表达式。防止搜索池被无意义的恒等变形（如 $E + (e - e)$）占满。
- **多邻域探索**：在细化过程中，我们为每个残差保留多个候选修正项，以探索到达相同精度的不同数学路径。

## 4. 性能与规模

### 并行处理

- **生成阶段**：每一层算式的笛卡尔积组合利用 **Rayon** 在多核并发处理。
- **细化阶段**：池中每个残差的定向搜索是完全独立的，在所有 CPU 核心上并发执行。

### 计算复杂度

- 深度为 $D$ 的标准 BFS 复杂度为 $O(B^D)$。
- 深度为 $M$、迭代 $N$ 次的块搜索（其中 $N \times M \approx D$）复杂度只有 $O(N \times B^M)$。

## 5. 使用指南

### 搜索工具 (Rust)

编译并运行:

```powershell
cargo run -r -- --target 114514 -m 5 -n 3 --constants e,pi,phi
```

#### 常用参数说明

- `-V, --target`: 目标数值（必填）。
- `-m, --max-ops`: 每个块的操作符数量。支持逗号分隔的列表（如 `4,3,2`），分别对应每次迭代的深度。如果只提供一个值，则所有迭代使用相同深度（默认 4）。
- `-n, --terms`: 迭代项数。例如 3 表示结果将形如 $A \pm B \pm C$（默认 3）。
- `-t, --top-k`: 每一轮保留的最优算式数量（默认 20）。
- `-c, --constants`: 初始常量池，支持 `e`, `pi`, `phi`, `sqrt2`, `ln2`（默认全选）。
- `-r, --results`: 最终打印出的结果数量（默认 5）。
- `-w, --workspace-size`: 搜索工作区大小（默认 5,000,000）。

### 分析工具 (Python)

使用 `analyze.py` 对搜索结果进行符号化简、LaTeX 渲染和高精度验证。

**环境依赖**：需要安装 `sympy`。

```powershell
pip install sympy
```

**运行方式**：

```powershell
python analyze.py "((pi^2)+e)"
```

如果不传入参数，脚本将运行其内部示例。

**功能说明**：

- **Original**: 原始算式。
- **Simplified**: 符号化简后的算式。
- **Expanded**: 全展开算式。
- **LaTeX**: 提供 Original、Simplified 和 Expanded 三种版本的 LaTeX 代码。
- **Value**: 提供 64 位精度的数值评估。
